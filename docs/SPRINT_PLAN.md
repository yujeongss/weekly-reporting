# 스프린트 계획

전략개발팀 주간보고 반자동화 프로젝트의 Phase별 스프린트 계획입니다.

---

## Phase 1 — 개인 탭 + 일감관리 + Excel 내보내기 `완료`

팀원 개인별 데이터 입력 UI와 Excel 내보내기 기능을 구축하는 초기 단계.

---

### Sprint 1: 기본 레이아웃 + 상태관리

**목표**: 전체 앱의 골격을 잡고 상태 관리 구조 확립

**주요 작업**
- 단일 HTML 파일 구조 설계 (CSS / HTML / JS 분리 없이 인라인)
- 팀원별 탭 레이아웃 구현
- 전역 `state` 객체 설계 및 localStorage 연동 초기화

**완료 기준**
- 팀원 탭 전환 시 각자의 데이터 영역이 독립적으로 표시됨
- 페이지 새로고침 후에도 탭 상태가 유지됨

---

### Sprint 2: 과제 카드 CRUD

**목표**: 진행 과제의 금주 실적 / 차주 계획 입력 및 관리

**주요 작업**
- 과제 카드 추가 / 수정 / 삭제 UI 구현
- 금주 실적 및 차주 계획 텍스트 필드 관리
- 입력 데이터 state 반영 및 localStorage 자동 저장

**완료 기준**
- 과제 카드 추가·삭제·수정이 정상 동작
- 새로고침 후 데이터가 복원됨

---

### Sprint 3: 운영/요청 + 제품 배포 CRUD

**목표**: 과제 외 운영·요청 업무와 제품 배포 항목 관리 기능 추가

**주요 작업**
- 운영/요청 항목 CRUD 구현
- 제품 배포 항목 CRUD 구현 (배포일, 제품명, 내용)
- 각 섹션별 빈 상태(empty state) 처리

**완료 기준**
- 세 가지 섹션(과제 / 운영·요청 / 제품 배포) 모두 CRUD 완성
- 팀원별로 독립된 데이터 유지

---

### Sprint 4: Excel export (from-scratch → template-based 전환)

**목표**: 기존 보고 양식 포맷을 완전히 재현하는 Excel 파일 생성

**주요 작업**
- ExcelJS CDN 연동 및 from-scratch export 초안 구현
- 포맷 깨짐 문제 확인 후 template-based 방식으로 전환
- 원본 `.xlsx`를 base64로 HTML에 내장, 로드 후 데이터 덮어쓰기 구현
- `clearSheetData()` 패턴으로 기존 병합/값 초기화 후 재작성

**완료 기준**
- 생성된 Excel 파일이 원본 양식 포맷(병합 셀, 폰트, 색상)을 유지함
- 입력한 과제 / 운영·요청 / 제품 배포 데이터가 정확한 셀에 삽입됨

---

## Phase 2 — 팀 시트 자동 취합 `예정`

팀원 개인 데이터를 하나의 팀 전체 시트로 자동 취합하는 단계.

---

### Sprint 5: 팀 시트 전체 구조 (금주실적/차주계획 자동 취합)

**목표**: 팀 탭에서 전체 팀원의 금주 실적 및 차주 계획을 자동으로 취합해 표시

**주요 작업**
- 팀 시트 탭 UI 추가
- 모든 팀원 state를 순회하여 금주 실적 / 차주 계획 취합 로직 구현
- 팀 시트 Excel export 섹션 구성

**완료 기준**
- 팀 탭에서 전 팀원의 금주 실적 및 차주 계획이 하나의 뷰에 표시됨
- Excel export 시 팀 시트 포함

---

### Sprint 6: 프로젝트 진척도 섹션

**목표**: 프로젝트별 전체 진척률을 팀 시트에 자동으로 집계

**주요 작업**
- 개인 탭에 프로젝트 진척도 입력 필드 추가
- 팀 시트에서 프로젝트별 진척도 취합 및 시각화
- Excel export에 진척도 섹션 반영

**완료 기준**
- 팀 시트 내 프로젝트 진척도 섹션이 자동 집계됨

---

### Sprint 7: 주요이슈/협조 섹션 + 제품배포 취합

**목표**: 주요 이슈 및 팀 간 협조 요청 항목과 전체 제품 배포 내역 취합

**주요 작업**
- 개인 탭에 주요이슈 / 협조요청 입력 필드 추가
- 팀 시트에서 주요이슈·협조 섹션 자동 취합
- 전체 팀원의 제품 배포 항목을 팀 시트에 통합

**완료 기준**
- 팀 시트에 주요이슈 / 협조요청 / 제품배포 섹션이 모두 자동 생성됨

---

## Phase 3 — AI 챗봇 패널 `예정`

Claude API를 연동해 보고서 초안 작성 및 요약을 AI가 지원하는 단계.

---

### Sprint 8: 우측 패널 AI chat UI

**목표**: 앱 우측에 AI 대화 패널 레이아웃 구성

**주요 작업**
- 우측 사이드 패널 슬라이드인/아웃 UI 구현
- 채팅 말풍선 컴포넌트 (사용자 / AI 구분)
- 입력창 및 전송 버튼 구현

**완료 기준**
- AI 패널이 열리고 닫히며, 메시지 입출력 UI가 동작함 (API 미연동 상태)

---

### Sprint 9: Claude API 연동

**목표**: 실제 Claude API 호출로 AI 응답 수신

**주요 작업**
- API 키 입력 UI 및 localStorage 저장 처리
- `fetch`를 통한 Claude API 호출 구현
- 스트리밍 응답 렌더링 (선택)

**완료 기준**
- 채팅창에서 메시지를 보내면 Claude의 실제 응답이 표시됨

---

### Sprint 10: 보고서 자동 요약/작성 지원

**목표**: 현재 입력된 데이터를 바탕으로 AI가 보고서 문구를 자동 생성

**주요 작업**
- state 데이터를 프롬프트로 변환하는 컨텍스트 빌더 구현
- "이번 주 보고서 요약해줘" 등 단축 명령어 버튼 추가
- AI 생성 문구를 해당 필드에 바로 적용하는 기능

**완료 기준**
- AI가 현재 데이터를 분석해 보고서 문구 초안을 제안함
- 제안 문구를 클릭 한 번으로 입력 필드에 적용 가능
